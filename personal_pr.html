• 지원동기

    팀 스파르타에서 진행하는 항해99에서 프론트엔드 과정을 배우던 중 백엔드와 협업 프로젝트를 몇 번 진행했었습니다. 프로젝트 과정 중 서버 교육 과정이 없어서 백엔드와의 협업 시 기술적 의사 결정에서 몇 번 마찰을 겪은 경험이 있습니다.
    이 경험을 통해 프론트엔드 개발자를 벗어나 웹 개발자를 목표로 한다면 백엔드 교육은 필수라는 생각이 들게 되어 이 과정에 지원하게 되었습니다.

  • 개발자를 선택하게 된 이유

    머릿속에서 상상만 가능했던 아이디어들을 실제로 구현할 수 있는
    개발자라는 직업에 매료되었습니다.
    어린 시절 저는 진로 희망 작성란에 항상 발명가를 적었었습니다.
    새로운 아이디어를 상상하고 직접 창조해내며 기존에 있던 아이템들의 단점을 개선하고 좀 더 편리하게 사용할 수 있도록 보완하는 일이 즐겁고 행복했습니다.
    진로를 고민하던 시기에 우연히 개발을 접하게 되었는데 어렸을 때 발명가를 꿈꾸며 느꼈던 감정과 같은 느낌을 받아 매우 흥미로웠습니다.
    특히 프론트엔드 개발은 제 상상 속 아이디어를 눈앞에 시각적으로 구현할 수 있고 누구나 사용 가능한 웹 환경을 만들어낼 수 있다는 점에서 큰 매력을 느꼈습니다. 자연스레 개발자가 제 장래희망이 되었고 꿈을 이루기 위해 끊임없이 학습하였습니다. 또한, 현직자들과의 인터뷰를 자기주도적으로 진행하여 현 개발 시장과 현업에 대한 지식도 습득하기 위해 노력하였습니다.

  • 기억에 남는 개발 프로젝트

    저에게 가장 인상 깊었던 프로젝트는 'Nine Cloud'라는 프로젝트입니다.
    이 프로젝트는 프론트엔드 3명, 백엔드 2명, 디자이너 1명이 협업하여 약 5~6주간 진행되었습니다. 모든 팀원들에게 이 프로젝트가 처음 참여하는 본격적인 프로젝트였기 때문에, 서로의 부족한 점들로 인해 많은 어려움을 겪었던 경험이 기억에 남습니다. 가장 어려움을 겪었던 부분을 손에 꼽자면 역시 백엔드와의 기능 구현 문제 관련 소통이었습니다. 서로의 기술 스택에 대한 정보가 부족하고 처음 구현해 보는 기능에 대해선 본인의 코드에 대한 확신이 없기 때문에 문제가 발생했을 때 오류가 생기는 부분을 찾는 과정에서 많은 시간을 소요했었습니다.
    이 부분을 개선하기 위해 백엔드 팀원과 본인의 코드 리뷰를 진행하며 서로의 기술 스택 이해도를 높여가는 시간을 가졌습니다. 또한, 밤을 새워 직접 백엔드 코드를 작성하여 문제를 조속히 해결하기 위해 노력하였습니다.
    결과적으로 서로에 대한 기술 이해도가 높아지고 프로젝트 과정이 매끄러워지는 성과를 얻었습니다.

  • 개발 프로젝트 진행 시 주요 담당 역할

    프론트엔드 개발자로 프로젝트 진행 시 클라이언트와 서버 간의 원활한 데이터 통신을 위해 Axios를 사용한 API 연동 작업을 주로 수행했습니다.
    API 명세서를 바탕으로 데이터 요청 및 응답 처리를 구현하고, 오류 발생 시 이를 해결하기 위해 백엔드 팀원들과 적극적으로 소통하였습니다.
    또한 실시간 데이터 처리가 요구되는 기능을 구현할 때도 연결 유지 및 데이터 송수신의 안정성을 확보하기 위해 백엔드 개발자들과 지속적으로 조율하며 최적의 솔루션을 도출했습니다.
    이 과정에서 자연스레 프론트엔드 개발자들과 백엔드 개발자들의 연결고리가 되어 서로간의 소통을 매끄럽게 만들어주는 윤활유 역할을 맡고 팀에 활력을 불어넣어 주기 위해 항상 적극적인 자세로 소통하여 협업에 임했습니다.
    이러한 노력으로 팀의 전반적인 커뮤니케이션이 원활해졌고 프로젝트의 진행 속도와 효율성도 크게 향상될 수 있었습니다. 또한 여러 경험을 바탕으로 발표나 홍보같은 개발 외의 활동도 적극적으로 자원하였습니다.

  • 다른 사람과 구별되는 나만의 강점

    다른 사람과 구별되는 나만의 강점은 역시 무엇이든 일단 시도해보는 부분이라 생각합니다. 유명한 브랜드 Nike의 'Just Do It' 슬로건을 모토로 삼아 새로운 일에 도전하는 것을 두려워 하지 않고 온전히 몰두하기 위해 노력합니다. 이러한 접근 방식 덕분에 프로젝트에서 새로운 기술을 구현해야 할 때마다 주저 없이 자원해 구현에 성공한 경험이 있습니다. 또한 현업에 궁금증이 생겼을 때 현직 개발자들을 직접 만나 인터뷰를 진행하여 실질적인 지식을 얻고 부족한 부분과 보완해야 할 부분들을 개선해 나가고 있습니다.

  • 성격의 장점과 단점

    제 성격의 가장 큰 장점은 친화력입니다. 사람들과 쉽게 친해지며 다양한 성격의 사람들과도 잘 어울립니다. 이러한 친화력을 이용하여 근무에 지친 팀원들을 격려하거나 분위기를 환기시키는 역할을 맡곤 합니다. 실제로 과거 근무했던 맥도날드에서 신입 크루들을 잘 이끌어주고 도와주었다는 이유로 인센티브를 받았던 경험과 부트캠프에서 가장 팀원들과 소통을 열심히 하고 힘들 때 많은 격려를 해줬던 팀원에 뽑혀 수상을 했던 경험이 있습니다. 단점으로는 때때로 꼼꼼함이 지나쳐 강박적으로 보일 수 있다는 점입니다. 제가 일을 할 때 매우 정확하게 처리하려는 경향이 있어 이로 인해 작업 속도가 느려졌던 경험이 있습니다. 이 점을 인식하고 개선하기 위해 첫 시도에 더욱 꼼꼼하게 검토하고 불안함을 지우기 위해 관련 기술을 향상시키기 위해 노력하고 있습니다.

  • 내가 생각하는 좋은 코드란

    코드 작성 규칙들에 위배되지 않으며 가독성이 뛰어나게 작성된 코드가 좋은 코드라고 생각합니다. 코드도 결국 글쓰기와 마찬가지로 규칙을 지키고 가독성이 뛰어난 글이 읽기에도 좋고 보기에도 좋듯이 코드도 똑같다고 생각합니다. 또한 차후에 유지보수도 훨씬 용이하게 할 수 있고 코드 리뷰과정을 원활하게 만들어줍니다. 더군다나 개인 프로젝트가 아닌 협업 상황에서는 이러한 특성이 더욱 중요한데, 모든 팀원이 일관된 코딩 스타일을 유지하고 가독성이 뛰어난 코드를 작성하면 프로젝트의 효율성이 크게 향상되고 팀 내 커뮤니케이션이 원활해지기 때문입니다.

  • 입사 후 포부

    배움은 남녀노소 가리지 않고 중요하다고 생각합니다. 현재 사회는 배움과 서로에 대한 이해가 부족하여 여러 문제가 발생하고 있습니다. 에듀테크 분야에 입사하여 여러 사람들에게 배움에 대한 중요성을 널리 알리고 싶습니다. 감사합니다!
